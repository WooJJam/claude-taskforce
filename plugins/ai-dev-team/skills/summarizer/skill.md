# Summarizer Skill

역할: Summarizer. Implementer, Reviewer, QA 결과를 받아 최종 리포트를 작성한다.

## 언제 이 스킬을 사용하나요?

- `/task` 워크플로우 Step 6 실행 시
- Implementer, Reviewer, QA가 모두 완료된 후

---

## 수행 작업

전달받은 구조화된 데이터를 기반으로 아래 형식의 최종 리포트를 작성한다.
요약이나 해석 없이 전달받은 데이터를 그대로 정리한다.

---

## 출력 형식

```
## 최종 요약 리포트 — run-id: {run-id}

---

### 구현 결과

**변경 파일**
| 파일 | 작업 |
|------|------|
| ...  | ...  |

**TDD 사이클**
| 케이스 | Red | Green | Refactor |
|--------|-----|-------|----------|
| ...    | ... | ...   | ...      |

---

### 코드 리뷰 결과

**[CRITICAL]**
- (없으면 "없음")

**[MAJOR]**
- (없으면 "없음")

**[MINOR]**
- (없으면 "없음")

---

### QA 결과

| 시나리오 | 입력 | 기대 결과 | 실제 결과 | 통과 여부 |
|---------|------|----------|----------|----------|
| ...     | ...  | ...      | ...      | ✅ / ❌  |

통과: N개 / 실패: M개

---

### 완료 상태

- [x/o] 승인된 계획 기준 구현
- [x/o] 전체 테스트 통과
- [x/o] 리뷰 CRITICAL/MAJOR 없음
- [x/o] QA 전체 통과
```
