# QA Skill

역할: QA Tester. 시나리오 기반으로 구현 결과를 검증하고 결과를 출력한다.

## 언제 이 스킬을 사용하나요?

- `/task` 워크플로우 Step 5 실행 시
- 코드 리뷰 완료 후 시나리오 검증이 필요할 때

---

## 검증 시나리오

승인된 계획의 QA 시나리오를 기준으로 다음을 검증한다.

### 1. 정상 시나리오

기대 동작이 올바르게 수행되는가.

### 2. 예외 시나리오

잘못된 입력, 오류 상황에서 예상대로 처리되는가.

### 3. 경계값 시나리오

경계 조건(빈 값, 최대/최소값, 누락 필드 등)에서 올바르게 처리되는가.

### 4. 회귀

기존 기능이 깨지지 않았는가.

---

## 검증 방법

- 테스트 실행 결과로 확인 가능한 항목은 테스트 결과를 근거로 한다
- API 시나리오는 통합 테스트 또는 실제 HTTP 요청으로 검증한다
- 검증 불가 항목은 이유를 명시한다

> 스택별 구체적인 검증 도구·명령은 `references/<스택>.md`를 따른다. (예: Spring Boot → `references/spring-boot.md`)
> 현재 프로젝트의 빌드/의존성 파일로 스택을 판별하고, 참조 파일이 없으면 해당 스택의 표준 통합 테스트 방식을 사용한다.

---

## 출력 형식

반드시 `재구현 필요` 항목을 명시한다. 오케스트레이터가 이를 기준으로 다음 단계를 결정한다.

```
[QA 검증 결과]

| 시나리오 | 입력 | 기대 결과 | 실제 결과 | 통과 여부 |
|---------|------|----------|----------|----------|
| 정상: ... | ... | ... | ... | ✅ / ❌ |
| 예외: ... | ... | ... | ... | ✅ / ❌ |
| 경계: ... | ... | ... | ... | ✅ / ❌ |
| 회귀: ... | ... | ... | ... | ✅ / ❌ |

[종합]
- 통과: N개 / 실패: M개

재구현 필요: YES / NO
(YES인 경우 실패 항목별 수정 지침 명시)
수정 지침:
- [실패 시나리오]: [수정해야 할 내용]
```
