# Planner Skill

역할: Planner. Researcher 조사 결과를 바탕으로 구현 계획을 작성한다.

## 언제 이 스킬을 사용하나요?

- `/task` 워크플로우 Step 2 실행 시
- 코드베이스 조사 완료 후 구현 계획이 필요할 때

---

## 수행 작업

다음 항목을 모두 포함한 구현 계획을 작성한다:

1. **구현 목표** — 이 작업이 달성하는 것
2. **수정 대상** — 변경할 파일 목록과 각 파일에서 할 작업
3. **구현 순서** — 작업 순서와 이유
4. **테스트 전략** — 단위/통합, 정상/예외/경계값 케이스 목록
5. **검증 명령** — 빌드, 테스트 실행 명령어
6. **QA 시나리오** — 시나리오별 입력과 기대 응답
7. **주요 리스크** — 예상 문제와 대응 방안

---

## 금지 사항

- 코드 파일 수정 금지
- 구현 목적의 Bash 명령 실행 금지

---

## 승인 게이트

계획 완료 후 run-id를 표시하고 승인을 요청한다:

> `APPROVED: ai-run-YYYY-MM-DD-NNN` 형식으로 승인해 주세요.
> 승인 전까지 구현을 시작하지 않습니다.

`APPROVED: <run-id>` 수신 전 코드 파일 수정 금지.
"ok", "진행해" 등 모호한 표현은 승인으로 인정하지 않는다.

---

## 출력 형식

반드시 아래 구조로 출력한다. 다음 에이전트가 이 형식을 그대로 입력으로 받는다.

```
[구현 계획] run-id: ai-run-YYYY-MM-DD-NNN

1. 구현 목표
   ...

2. 수정 대상
   | 파일 경로 | 작업 내용 |
   |----------|----------|
   | ...      | ...      |

3. 구현 순서
   1. ...
   2. ...

4. 테스트 전략
   - 단위 테스트: ...
   - 통합 테스트: ...
   - 케이스: 정상(...), 예외(...), 경계값(...)

5. 검증 명령
   ...

6. QA 시나리오
   | 시나리오 | 입력 | 기대 결과 |
   |---------|------|----------|
   | ...     | ...  | ...      |

7. 주요 리스크
   - ...
```
